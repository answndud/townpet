# TownPet Impeccable Context

This repository uses repo-local Impeccable setup only.

- Installed skill path: `.agents/skills/impeccable`
- Strategic product context: `PRODUCT.md`
- Visual design context: `DESIGN.md`
- Detector scripts: `pnpm -C app design:detect` and `pnpm -C app design:detect:json`

Current Impeccable `.agents` skill versions load `PRODUCT.md` and `DESIGN.md` through `.agents/skills/impeccable/scripts/load-context.mjs`. Keep this file as a human-readable compatibility pointer, but treat `PRODUCT.md` and `DESIGN.md` as the active context files.

## Design Context Summary

TownPet is a product UI, not a marketing-first site. It should feel calm, structured, and neighborly: a reliable local information surface for pet owners and a dense operating surface for moderators.

Future frontend work should respect existing `tp-*` utilities, Tailwind class conventions, blue-tinted surfaces, compact controls, Space Grotesk typography, and IBM Plex Mono metadata use. Improve screens by strengthening locality, trust, policy visibility, scan paths, loading/empty/error states, and mobile fit.

Avoid generic AI design patterns: purple-blue gradients, nested card grids, glassmorphism, gradient text, decorative mascot overload, and hero-style marketing layouts on functional product screens.
