# 01. 지표/KPI 설계

## 1) MVP 핵심 지표
### Local
- 7일 리텐션(동네 기준 코호트)
- 템플릿 완성률(방문일/항목/비용/대기 입력률)
- 저장률(저장/조회)
- 검색 사용률(검색/DAU)

### Market/돌봄
- 문의율(조회 대비 문의)
- 성사율(문의 대비 완료)
- 분쟁/신고율(거래/요청 단위)

### Lost&Found
- 알림 도달→제보 전환율
- 평균 제보 발생 시간(골든타임)
- 허위 신고율

### Global
- 질문당 답변 수
- 답변 채택률
- 검색 유입(자연 검색)

## 3) 첫 90일 획득 이벤트 사전

TownPet의 첫 90일 획득 루프는 `방문 -> 동네/주제 탐색 -> 글쓰기 템플릿 진입 -> 첫 글/댓글 -> 7일 재방문 -> 공유`를 기준으로 본다.

클라이언트 이벤트 수집은 `NEXT_PUBLIC_ENABLE_CLIENT_TELEMETRY=1`일 때만 켜진다. public 랜딩/가이드 이벤트는 개인 식별 로그를 남기지 않고 `AcquisitionEventStat` 일별 집계로만 저장한다.

| 이벤트 | 의미 | 현재 연결 surface | 주요 차원 |
|---|---|---|---|
| `LANDING_VIEWED` | `/` 첫 랜딩 조회 | `HOME` | `targetId=home` |
| `FEED_CTA_CLICKED` | 전체/주제 피드로 이동 | `HOME`, `CAMPAIGN_NEIGHBORHOOD_MAP`, `TOWN_LANDING`, `TOWN_SECTION` | CTA 위치 또는 지역/섹션 |
| `ONBOARDING_CTA_CLICKED` | 내 동네 설정 시작 | `HOME`, `CAMPAIGN_NEIGHBORHOOD_MAP`, `TOWN_LANDING` | CTA 위치 또는 지역 |
| `CAMPAIGN_VIEWED` | 지도 만들기 캠페인 조회 | `CAMPAIGN_NEIGHBORHOOD_MAP` | `targetId=neighborhood_map` |
| `CAMPAIGN_CTA_CLICKED` | 캠페인 참여/템플릿 CTA 클릭 | `CAMPAIGN_NEIGHBORHOOD_MAP` | CTA 또는 템플릿 라벨 |
| `GUIDE_VIEWED` | SEO guide 조회 | `GUIDE` | guide slug |
| `GUIDE_CTA_CLICKED` | guide에서 피드/작성/다른 guide 이동 | `GUIDE` | guide slug + CTA 위치 |
| `TOWN_LANDING_VIEWED` | 사용자 선택 동네 허브 조회 | `TOWN_LANDING` | town slug |
| `TOWN_CATEGORY_CLICKED` | 동네 허브 카테고리 클릭 | `TOWN_LANDING`, `TOWN_SECTION` | town slug + section slug |
| `TOWN_SECTION_VIEWED` | 동네 섹션 페이지 조회 | `TOWN_SECTION` | town slug + section slug |
| `WRITE_TEMPLATE_OPENED` | 글쓰기 템플릿 진입 | `POST_CREATE`, `TOWN_LANDING`, `TOWN_SECTION` | template id 또는 post type |
| `LOST_ALERT_CREATED` | 분실동물 글 생성 | `LOST_FLOW` | post type 또는 town |
| `LOST_POSTER_GENERATED` | 분실 포스터 이미지 생성 | `SHARE_PANEL` | post id |
| `LOST_SIGHTING_CREATED` | 목격 제보 생성 | `LOST_FLOW` | post id |
| `KAKAO_SHARE_CLICKED` | 카카오 공유 클릭 | `SHARE_PANEL` | post id 또는 guide slug |
| `OPERATOR_CONTENT_VIEWED` | 운영자 정리 콘텐츠 조회 | `GUIDE`, `TOWN_SECTION` | 콘텐츠/섹션 |
| `CORRECTION_REQUEST_SUBMITTED` | 병원/장소 정보 정정 요청 제출 | `CORRECTION_FLOW` | post id |
| `FOUNDING_MEMBER_JOINED` | 창립 멤버 참여/부여 | `ADMIN_OPS` | user or campaign |

## 4) 첫 90일 퍼널 판정

| 퍼널 단계 | 대표 이벤트 | 같이 볼 제품 지표 |
|---|---|---|
| 방문 | `LANDING_VIEWED`, `GUIDE_VIEWED`, `CAMPAIGN_VIEWED`, `TOWN_LANDING_VIEWED` | 유입 채널별 landing 조회, guide별 CTA율 |
| 동네/주제 탐색 | `TOWN_CATEGORY_CLICKED`, `FEED_CTA_CLICKED` | 지역별 카테고리 클릭률, 피드 진입률 |
| 첫 참여 준비 | `WRITE_TEMPLATE_OPENED`, `ONBOARDING_CTA_CLICKED` | 템플릿 진입률, 가입/온보딩 전환 |
| 첫 행동 | `post_created`, `lost_alert_created`, `lost_sighting_created`, `comment_created` | 첫 글/댓글 전환, 분실 제보 전환 |
| 재방문 | D7 세션/방문 코호트 | 첫 행동 후 7일 재방문 |
| 공유/확산 | `KAKAO_SHARE_CLICKED`, `lost_poster_generated` | 분실/가이드 공유 클릭률 |

P1-4 구현 범위에서는 방문/CTA/템플릿 진입 이벤트를 앱에 연결했다. 글 생성, 댓글, 공유, 정정 요청처럼 이미 서버 mutation이 있는 이벤트는 다음 phase에서 action/service 경계에 붙인다.
