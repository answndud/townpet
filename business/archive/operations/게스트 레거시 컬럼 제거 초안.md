# Guest Legacy Column Drop Draft

Updated: 2026-02-25

## Scope

Drop the following legacy columns after GuestAuthor migration is stable:

- `Post.guestDisplayName`
- `Post.guestIpDisplay`
- `Post.guestIpLabel`
- `Post.guestPasswordHash`
- `Post.guestIpHash`
- `Post.guestFingerprintHash`
- `Comment.guestDisplayName`
- `Comment.guestIpDisplay`
- `Comment.guestIpLabel`
- `Comment.guestPasswordHash`
- `Comment.guestIpHash`
- `Comment.guestFingerprintHash`

## Preconditions (must be green)

1. `pnpm db:verify:guest-authors` passes.
2. `GUEST_LEGACY_CLEANUP_STRICT=1 pnpm db:check:guest-legacy-cleanup` passes.
3. `pnpm db:rehearse:guest-legacy-cleanup` passes.
4. Quality gate green on latest main.

## Draft SQL

```sql
ALTER TABLE "Post" DROP COLUMN "guestDisplayName";
ALTER TABLE "Post" DROP COLUMN "guestIpDisplay";
ALTER TABLE "Post" DROP COLUMN "guestIpLabel";
ALTER TABLE "Post" DROP COLUMN "guestPasswordHash";
ALTER TABLE "Post" DROP COLUMN "guestIpHash";
ALTER TABLE "Post" DROP COLUMN "guestFingerprintHash";

ALTER TABLE "Comment" DROP COLUMN "guestDisplayName";
ALTER TABLE "Comment" DROP COLUMN "guestIpDisplay";
ALTER TABLE "Comment" DROP COLUMN "guestIpLabel";
ALTER TABLE "Comment" DROP COLUMN "guestPasswordHash";
ALTER TABLE "Comment" DROP COLUMN "guestIpHash";
ALTER TABLE "Comment" DROP COLUMN "guestFingerprintHash";
```

## Rollout notes

- Run in maintenance window or low-traffic window.
- Execute db backup before applying DDL.
- After apply:
  - run smoke checks (`/feed`, post detail, comment create/edit/delete)
  - run `pnpm quality:check`
  - monitor error rate for `GUEST_*` service errors

## Rollback strategy

- Primary: restore from DB backup if unexpected regression appears.
- Secondary: rollback app deployment and replay recovery migration if needed.
