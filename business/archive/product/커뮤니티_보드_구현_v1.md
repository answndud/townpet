# community-board-implementation-v1.md
# 커뮤니티/공용보드 구현 설계 (v1)

기준일
- 2026-02-25

문서 목적
- 멀티종 커뮤니티 구조에서 `커뮤니티 전용 보드`와 `공용 보드`를 동시에 운영하기 위한 구현 단위를 정의한다.
- `병원후기`, `실종/목격`, `중고/공동구매`를 공용 보드로 고정하고 나머지 카테고리는 커뮤니티 중심으로 운영한다.

관련 문서
- `business/product/community-taxonomy-v1.md`

---

## 1) 목표/비목표

목표
- 사용자 입장에서 카테고리 선택 시 어디에 노출되는지(커뮤니티/공용)를 명확히 만든다.
- 기존 `PostType` 체계와 안전정책을 최대한 재사용한다.
- 라우팅/쿼리/검증 규칙을 단순하고 설명 가능한 형태로 유지한다.

비목표
- 사용자 커뮤니티 생성/승인 플로우
- 커뮤니티 멤버십/가입/구독 기능
- 공용 보드 별도의 완전 신규 작성 폼(기존 작성 폼 확장으로 처리)

## 2) 데이터 모델 초안 (Prisma)

### 2.1 신규 enum

```prisma
enum BoardScope {
  COMMUNITY
  COMMON
}

enum CommonBoardType {
  HOSPITAL
  LOST_FOUND
  MARKET
}
```

### 2.2 신규 모델

```prisma
model CommunityCategory {
  id          String      @id @default(cuid())
  slug        String      @unique
  labelKo     String
  description String?
  sortOrder   Int         @default(100)
  isActive    Boolean     @default(true)
  communities Community[]
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
}

model Community {
  id               String            @id @default(cuid())
  categoryId       String
  slug             String            @unique
  labelKo          String
  description      String?
  isActive         Boolean           @default(true)
  sortOrder        Int               @default(100)
  defaultPostTypes PostType[]
  tags             String[]
  category         CommunityCategory @relation(fields: [categoryId], references: [id])
  posts            Post[]
  createdAt        DateTime          @default(now())
  updatedAt        DateTime          @updatedAt

  @@index([categoryId, sortOrder])
}
```

### 2.3 Post 확장

```prisma
model Post {
  // 기존 필드 생략
  boardScope      BoardScope      @default(COMMUNITY)
  petTypeId       String?         @map("communityId")
  commonBoardType CommonBoardType?
  animalTags      String[]

  petType         Community?      @relation(fields: [petTypeId], references: [id])

  @@index([boardScope, createdAt(sort: Desc)])
  @@index([petTypeId, createdAt(sort: Desc)], map: "Post_communityId_createdAt_idx")
  @@index([commonBoardType, createdAt(sort: Desc)])
}
```

제약 규칙(애플리케이션 레벨)
- `boardScope=COMMUNITY` -> `petTypeId` 필수, `commonBoardType` null
- `boardScope=COMMON` -> `commonBoardType` 필수, `petTypeId` null 허용
- `boardScope=COMMON`에서 `animalTags` 최소 1개 권장(탐색성 확보)

## 3) 카테고리-보드 매핑 규칙

강제 매핑(작성 시)
- `HOSPITAL_REVIEW` -> `boardScope=COMMON`, `commonBoardType=HOSPITAL`
- `LOST_FOUND` -> `boardScope=COMMON`, `commonBoardType=LOST_FOUND`
- `MARKET_LISTING` -> `boardScope=COMMON`, `commonBoardType=MARKET`

기본 매핑(작성 시)
- `FREE_BOARD`, `QA_QUESTION`, `PRODUCT_REVIEW`, `PET_SHOWCASE`, `PLACE_REVIEW`, `WALK_ROUTE`, `MEETUP`
  -> `boardScope=COMMUNITY`, `petTypeId` 필수

레거시 매핑
- `FREE_POST`, `DAILY_SHARE` -> `FREE_BOARD` 그룹으로 간주
- `QA_ANSWER` -> `QA_QUESTION` 그룹으로 간주

## 4) API 계약 초안

### 4.1 커뮤니티 디렉토리/상세

`GET /api/communities`
- query: `category?`, `q?`, `limit?`, `cursor?`
- response: `{ ok: true, data: { items, nextCursor } }`

`GET /api/communities/:slug`
- response: `{ ok: true, data: { community, stats } }`

### 4.2 공용 보드 목록

`GET /api/boards/:commonBoardType/posts`
- `commonBoardType`: `hospital | lost-found | market`
- query: `scope?`, `q?`, `animalTag?`, `limit?`, `cursor?`, `sort?`
- response: `{ ok: true, data: { items, nextCursor } }`

### 4.3 게시글 생성/수정

`POST /api/posts`
- body 공통: 기존 + `boardScope`, `petTypeId?`, `commonBoardType?`, `animalTags?`
- 서버 규칙:
  - `type` 기반으로 `boardScope/commonBoardType`를 강제 재계산(클라이언트 입력 신뢰 금지)
  - 공용 보드는 `animalTags` sanitize + max count 제한 적용

`PATCH /api/posts/:id`
- 카테고리 변경 시 매핑 규칙 재검증
- 이미 생성된 공용 보드 글의 `commonBoardType`은 타입 변경 외 직접 수정 금지

## 5) 검증(Zod) 규칙 초안

`postCreateSchema` 확장
- `petTypeId`: optional string(cuid)
- `animalTags`: optional array(min 1, max 5, each max length 24)

Refine
- `type`이 공용 3종이면 `petTypeId`가 없어야 한다.
- `type`이 공용 3종이 아니면 `petTypeId`가 있어야 한다.

에러 코드 권장
- `POST_COMMUNITY_REQUIRED`
- `POST_COMMUNITY_FORBIDDEN`
- `POST_COMMON_BOARD_TAGS_REQUIRED`

## 6) UI/IA 변경 초안

### 6.1 상단 탐색
- `커뮤니티` 탭: `/communities`
- `공용보드` 탭 묶음:
  - 병원후기 `/boards/hospital`
  - 실종/목격 `/boards/lost-found`
  - 중고/공동구매 `/boards/market`

### 6.2 글쓰기
- Step 1: 카테고리 선택
- Step 2:
  - 공용 카테고리 선택 시: 공용 보드 자동 선택 + `animalTags` 입력
  - 커뮤니티 카테고리 선택 시: 커뮤니티 선택 필수

### 6.3 피드
- 기본 피드는 기존 유지
- 필터 추가: `source=community|common|all`

## 7) 마이그레이션/릴리스 순서

1. Schema expand: enum/model/field 추가
2. Seed: L1/L2 커뮤니티 고정 데이터 삽입
3. Read path: 커뮤니티/공용보드 조회 API 추가
4. Write path: post create/update 매핑 강제
5. UI: 커뮤니티 디렉토리 + 공용 보드 라우트 + 글쓰기 분기
6. Cleanup: 레거시 타입 그룹 정리(선택)

롤백 전략
- `boardScope` 기본값을 `COMMUNITY`로 유지해 기존 글 읽기 호환 확보
- 공용보드 신설 후에도 기존 `/feed` 조회 경로 fallback 유지
