# 03. 마켓/대여/나눔 정책(초안)

## 1) 중고
- 금지 품목: 불법/규제 품목, 동물 생체 판매(원칙적으로 금지), 위조품
- 상태 표기 의무: 새상품급/상/중/하 + 하자 설명
- 거래 상태: AVAILABLE/RESERVED/SOLD/CANCELLED

## 2) 대여(분쟁 최소화가 핵심)
- 대여 필수 정보: 대여료, 보증금, 기간, 반납 방식, 하자 기준
- 인수/반납 체크리스트
  - 인수 사진(권장) / 반납 사진(권장)
  - 오염/파손 기준 템플릿(표준 문구 제공)
- 연체 규정(초안)
  - 연체 시 추가 요금(정액/일 단위) — 결제 도입 후 적용 권장

## 3) 나눔
- 나눔은 “대가 요구 금지”를 기본으로 한다.
- 예외: 배송비 실비 수준의 요청만 허용(명시)

## 4) 스팸/업자 제한
- 마켓 글 작성 조건(초안)
  - 동네 설정 완료 + 가입 24시간 + 활동 점수 조건
- 외부 링크/연락처는 단계적으로 허용(초기에는 제한 권장)

## 5) Phase 2 상태 머신 preflight 결정

현재 코드에는 `MarketListing` 모델과 `MarketStatus` enum이 있지만, 작성 validation/service/UI와 조회 include에는 아직 연결되지 않았다. 따라서 상태 변경 액션보다 먼저 마켓 글이 구조화 레코드를 실제로 생성하고 읽는지 고정한다.

### M1. 구조화 마켓 글 생성/조회
- 대상: `MARKET_LISTING` 작성 시 `MarketListing` 생성
- 입력: `listingType`, `price`, `condition`, `depositAmount`, `rentalPeriod`
- 기본 상태: `AVAILABLE`
- 조회: feed/detail에서 가격, 거래 유형, 상태를 표시
- 정책: 신규 유저 제한, 비회원 작성 제한, 연락처 제한, 신고/자동숨김은 기존 post write/report 정책을 그대로 재사용
- 완료 기준: validation/service/query/UI 테스트가 존재하고, 마켓 글이 구조화 필드 없이 단순 자유글로만 저장되지 않는다.

### M2. 상태 전환 액션
- 상태 전환:
  - 작성자: `AVAILABLE -> RESERVED`, `RESERVED -> AVAILABLE`, `AVAILABLE/RESERVED -> SOLD`, `AVAILABLE/RESERVED -> CANCELLED`
  - 관리자: 모든 상태에서 필요한 수동 복구/취소 가능
  - 비작성자/게스트: 전환 불가
- 감사 로그: 상태 변경은 `ModerationActionLog`에 남긴다. 현재 enum에는 마켓 상태 변경 action이 없으므로 schema 확장이 필요하다.
- 분쟁 연결: 신고 사유 `FRAUD`는 관리자 큐에서 마켓 상태와 함께 보이게 한다.
- 제외: 결제, 정산, 배송, 외부 채팅, 에스크로
