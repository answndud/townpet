# 검색엔진 제출 체크리스트

목표: TownPet public page가 네이버 서치어드바이저와 일반 검색 로봇에 일관되게 수집되도록 `robots.txt`, sitemap, canonical, URL 검사 절차를 고정한다.

이 문서는 새 public route, guide, 지역 허브, SEO landing을 추가했거나 production 배포 후 검색 수집 상태를 점검할 때만 실행한다.

## 기준 URL

- Production origin: `https://townpet.vercel.app`
- robots: `https://townpet.vercel.app/robots.txt`
- sitemap: `https://townpet.vercel.app/sitemap/0.xml`
- 대표 제출 URL:
  - `https://townpet.vercel.app/`
  - `https://townpet.vercel.app/feed`
  - `https://townpet.vercel.app/boards/adoption`
  - `https://townpet.vercel.app/guides/lost-dog-poster`
  - `https://townpet.vercel.app/guides/24h-vet-checklist`
  - `https://townpet.vercel.app/guides/pet-used-trade-safety`
  - `https://townpet.vercel.app/guides/lost-pet-first-24-hours`
  - `https://townpet.vercel.app/guides/pet-hospital-review-policy`

## 소스 기준

- 네이버 서치어드바이저 robots.txt 가이드는 robots.txt가 사이트 루트에서 `text/plain`으로 2xx 응답해야 하며, sitemap 위치를 robots.txt에 기록하는 방식을 안내한다.
- 네이버 서치어드바이저 RSS 및 사이트맵 제출 가이드는 sitemap URL의 도메인이 소유확인 된 사이트와 같아야 하고, sitemap은 10MB 미만, 단일 sitemap은 50,000 URL 미만이어야 한다고 안내한다.
- 네이버 서치어드바이저 선호 URL 가이드는 동일 콘텐츠의 대표 URL을 canonical로 지정하고, URL은 상대 경로가 아니라 절대 경로로 표현하라고 안내한다.
- 네이버 서치어드바이저 URL 검사 가이드는 개별 URL의 수집, 색인 가능 여부, Meta, SEO 상태를 확인하는 절차를 제공한다.

## 배포 전 자동 확인

아래 명령은 코드 변경 후 커밋 전 실행한다.

```bash
corepack pnpm@9.12.3 -C app test -- src/app/robots.test.ts src/app/sitemap.test.ts src/app/guides/page.test.tsx src/lib/post-page-metadata.test.ts
corepack pnpm@9.12.3 -C app lint
corepack pnpm@9.12.3 -C app typecheck
corepack pnpm@9.12.3 -C app build
node scripts/refresh-docs-index.mjs --check
```

통과 기준:

- `/robots.txt`가 `/sitemap/0.xml`을 노출한다.
- sitemap 첫 페이지가 public landing, feed, adoption, guide route를 포함한다.
- sitemap에 login, register, profile, notifications, admin, API, 호환 redirect alias가 포함되지 않는다.
- guide와 public post metadata가 canonical, title, description을 가진다.
- build output에서 guide route가 SSG로 생성된다.

## 배포 후 production 확인

```bash
OPS_BASE_URL=https://townpet.vercel.app corepack pnpm@9.12.3 -C app ops:check:health
curl -fsSI https://townpet.vercel.app/robots.txt
curl -fsSL https://townpet.vercel.app/robots.txt
curl -fsSI https://townpet.vercel.app/sitemap/0.xml
curl -fsSL https://townpet.vercel.app/sitemap/0.xml | rg "guides/(lost-dog-poster|24h-vet-checklist|pet-used-trade-safety|lost-pet-first-24-hours|pet-hospital-review-policy)"
```

통과 기준:

- health check가 200과 `status: ok`를 반환한다.
- robots와 sitemap이 2xx로 응답한다.
- robots에 `Sitemap: https://townpet.vercel.app/sitemap/0.xml`이 있다.
- sitemap에 guide 5개가 모두 있다.
- sitemap 응답이 지나치게 느리거나 10MB에 가까워지지 않는다.

## 네이버 서치어드바이저 수동 제출 순서

1. 네이버 서치어드바이저에 `https://townpet.vercel.app`을 HTTPS 사이트로 등록하고 소유 확인을 완료한다.
2. `robots.txt` 도구에서 `https://townpet.vercel.app/robots.txt` 수집 및 검증을 실행한다.
3. robots 검사에서 아래 public URL이 허용되는지 확인한다.
   - `/`
   - `/feed`
   - `/boards/adoption`
   - `/guides/24h-vet-checklist`
4. robots 검사에서 아래 private URL이 차단 또는 noindex 정책 대상인지 확인한다.
   - `/admin`
   - `/api/health`
   - `/login`
   - `/profile`
   - `/notifications`
5. 사이트맵 제출 메뉴에 `https://townpet.vercel.app/sitemap/0.xml`을 제출한다.
6. URL 검사에서 대표 제출 URL 3개 이상을 확인한다.
   - `/`
   - `/guides/24h-vet-checklist`
   - `/guides/lost-pet-first-24-hours`
7. 사이트 상태 리포트가 갱신되는 1-2일 뒤 HTTPS, robots, sitemap, Meta, SEO 경고를 확인한다.

## canonical 정책

- public canonical:
  - `/`
  - `/feed`
  - `/boards/adoption`
  - `/guides/{guideSlug}`
  - `/posts/{postId}` 중 guest-readable active GLOBAL post
- canonical alias:
  - `/feed/guest`와 `/search/guest`는 sitemap에 넣지 않고 canonical은 `/feed`로 수렴한다.
  - query filter URL은 사용자가 탐색할 수 있지만 sitemap 기본 제출 대상은 아니다.
- noindex 또는 crawler 차단 대상:
  - `/admin`, `/api`, `/login`, `/register`, `/onboarding`, `/password`, `/profile`, `/notifications`, `/bookmarks`, `/saved`, `/my-posts`, `/users`
  - 존재하지 않는 town/guide/post route

## 변경 시 체크포인트

새 public route를 추가할 때:

- [ ] route에 `metadata` 또는 `generateMetadata`가 있다.
- [ ] canonical이 의도한 대표 URL로 수렴한다.
- [ ] sitemap 포함 여부를 명시적으로 결정한다.
- [ ] private/auth/admin/API surface가 sitemap에 들어가지 않는 테스트가 유지된다.
- [ ] 404/noindex 경로가 검색 색인 대상으로 남지 않는다.
- [ ] active 상태는 `docs/PLAN.md`, 완료 요약은 `docs/DONE.md`를 작업 상태에 맞게 갱신한다.

## 장애와 대응

| 증상 | 확인 | 대응 |
| --- | --- | --- |
| robots가 HTML로 응답 | `curl -fsSI /robots.txt` | Next route와 middleware를 확인하고 `robots.test.ts`를 보강한다. |
| sitemap 500 또는 DB 오류 | `curl -fsSL /sitemap/0.xml` | DB unavailable fallback이 작동하는지 `sitemap.test.ts`를 먼저 확인한다. |
| private route가 sitemap에 포함 | `src/app/sitemap.test.ts` | staticRoutes와 post where 조건을 수정한다. |
| guide가 검색에 안 잡힘 | 서치어드바이저 URL 검사 | sitemap 제출 상태, canonical, title/description, robots 허용 여부를 확인한다. |
| 삭제할 URL이 계속 노출 | URL 검사, HTTP status | noindex, 404/410, sitemap 제거 순서로 처리하고 robots 차단은 검색 제외 확인 후 적용한다. |
