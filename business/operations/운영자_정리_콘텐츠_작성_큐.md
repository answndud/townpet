# 운영자 정리 콘텐츠 작성 큐

목적: production demo content를 제거한 뒤, 샘플이 아닌 실제 운영자 확인 콘텐츠로 첫 화면과 지역 허브의 빈 느낌을 줄인다.

이 문서는 DB에 자동으로 글을 넣는 seed 계획이 아니다. 운영자가 출처를 확인하고 `/posts/new`에서 `운영자 정리 콘텐츠` 체크박스, 출처 이름, 출처 URL, 최종 확인일을 남겨 작성하는 큐다.

첫 7개 전국 공통 글의 바로 작성 가능한 초안은 [운영자_정리_콘텐츠_초안_팩.md](./운영자_정리_콘텐츠_초안_팩.md)를 우선한다.

## 원칙

- 운영 지역을 하나로 제한하지 않는다. 전국 공통으로 유효한 글은 지역명을 붙이지 않고, 지역 맥락이 필요한 글은 작성 시 운영자가 선택한 동네만 붙인다.
- 특정 지역을 launch 기본값, 실험 지역, 대표 지역처럼 고정해 public UI에 노출하지 않는다.
- 운영자 정리 콘텐츠는 사용자 경험담처럼 쓰지 않는다.
- 병원은 순위, 추천, 평점, “좋다/나쁘다” 판단을 쓰지 않는다.
- 의료 판단, 법적 판단, 특정 업체 비방, 가격 확정 표현을 피한다.
- 모든 글은 출처 이름, 출처 URL, 최종 확인일을 포함한다.
- 작성 후 방문/신청/연락 전에는 원문과 업체에 다시 확인하라는 문구를 포함한다.
- 분실동물 관련 글에는 전화번호, 이메일, 오픈채팅, 카카오톡 ID, 도로명·번지 주소 입력을 유도하지 않는다.

## 작성 전 체크리스트

1. 글이 전국 공통 안내인지, 특정 동네 맥락이 필요한 글인지 구분한다.
2. 출처가 공식/공개 자료인지 확인한다.
3. 원문 URL이 열리고 최신 상태인지 확인한다.
4. 글 유형을 고른다.
   - `HOSPITAL_REVIEW`: 병원 경험담이 아니라 운영자 확인 안내일 때도 병원/의료 시설 관련 구조화 필드가 필요하면 사용한다.
   - `WALK_ROUTE`: 산책 코스, 공원, 보행 조건 정리.
   - `LOST_FOUND`: 실제 실종 제보가 아닌 가이드/운영 안내는 피하고, 분실/목격 가이드는 guide route를 우선한다.
   - `FREE_BOARD` 또는 `FREE_POST`: 지역 공지, 체크리스트, 참여 안내.
5. `운영자 정리 콘텐츠`를 체크하고 출처 정보를 입력한다.
6. 게시 전 금지 표현을 확인한다.

## 카테고리별 작성 기준

### 병원

- 좋은 예:
  - “방문 전 전화로 확인할 항목”
  - “야간/응급 진료 여부는 변동될 수 있습니다”
  - “경험 공유이며 진단·처방 판단이 아닙니다”
- 금지:
  - “최고”, “추천 1위”, “가장 저렴”, “무조건 가능”
  - 특정 병원 비방 또는 의료 과실 단정
  - 검증되지 않은 가격/진료 가능 여부 확정

### 산책

- 좋은 예:
  - “대형견 적합 여부”
  - “혼잡 시간”
  - “목줄 주의 구간”
  - “물 마실 곳, 배변봉투함”
  - “차량/자전거 주의 구간”
- 금지:
  - 집 앞 위치를 추정할 수 있는 시작점/종료점
  - 야간에 위험할 수 있는 정밀 위치 공유
  - 사유지 무단 출입을 유도하는 표현

### 분실동물

- 좋은 예:
  - “동네·공원·역·상가명 수준”
  - “보호자 공개 제보는 필요한 경우에만”
  - “상세주소/연락처는 공개하지 않기”
- 금지:
  - 전화번호, 이메일, 오픈채팅, 카카오톡 ID 공개 유도
  - 도로명·번지 주소 공개 유도
  - 특정인을 의심하거나 책임자로 단정

## 운영 루틴

- 매일 1개 작성 전 10분:
  - 전날 `/api/home/feed`, `/feed/guest`, `/search/guest`에서 빈 주제를 확인한다.
  - 같은 주제의 중복 글이 있으면 새 글 대신 기존 글을 업데이트하거나 댓글로 보강한다.
- 게시 후 10분:
  - 제목, 출처 패널, 최종 확인일, 정정 요청 CTA를 확인한다.
  - public 화면에 `운영자 정리` 배지가 보이는지 확인한다.
- 매주 1회:
  - 조회/클릭/제보 전환을 보고 다음 주 슬롯을 교체한다.

## 완료 기준

첫 14일 큐가 성공하려면 아래를 만족해야 한다.

- 운영자 정리 글 14개 중 병원/산책/분실 관련 글이 10개 이상이다.
- 모든 글에 출처 이름, 출처 URL, 최종 확인일이 있다.
- 병원 글은 추천/순위가 아니라 확인 항목 중심이다.
- 산책 글은 개인정보와 정밀 위치 노출을 피한다.
- 분실동물 글은 공개 연락처와 상세주소를 요구하지 않는다.
- 최소 3개 글은 guide 또는 `/posts/new` 템플릿으로 이어진다.

## 다음 확장

- 전국 공통 글 7개를 먼저 작성한 뒤, 사용자가 실제로 선택한 동네/검색 유입/제보 요청이 생긴 지역부터 같은 템플릿을 지역 맥락에 맞게 복제한다.
- 운영자 정리 글이 10개 이상 쌓인 뒤에야 오프라인 QR/파트너 제안을 다시 시작한다.
- 병원/장소 엔티티 canonicalization은 별도 기능 phase로 분리한다.
