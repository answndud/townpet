# free-ops-weekly-checklist.md

# 무료 운영 체크리스트 (1인 운영 2-3분 루틴)

목표
- 외부 SaaS를 최소화한 상태에서, 운영자가 혼자서도 지속 가능한 최소 생존 확인을 수행한다.

원칙
- 기본 모니터링: `GitHub Actions + /api/health + Vercel 로그`
- CSP 위반, 성능 snapshot, prewarm은 기본 루틴이 아니라 필요할 때만 실행한다.
- Sentry는 선택사항(기본 OFF)
- 평소에는 2-3분 안에 끝낸다.
- 인증, 관리자, 신고, 정책, DB migration, env 변경이 있을 때만 관리자 화면 수동 확인을 추가한다.

---

## 0-1분: 배포 상태 확인

1. Vercel 프로젝트의 최근 배포 1건 확인
2. 실패 배포가 있으면 빌드 로그의 첫 에러 원인만 기록

완료 기준
- 최근 배포가 `Ready`이거나, 실패 원인이 메모되어 있음

---

## 1-2분: Health 스모크 실행

로컬 solo evidence runner 권장:
- `OPS_BASE_URL=https://townpet.vercel.app pnpm -C app ops:evidence:solo`
- 결과 파일: `docs/reports/ops-evidence-*.md`

GitHub Actions에서 아래 워크플로우 실행
- Workflow: `ops-smoke-checks`
- 입력
  - `target_base_url`: `https://townpet.vercel.app`
  - `verify_sentry`: `false` (기본값)

완료 기준
- `ops:evidence:solo` status PASS 또는 `Check deployment health endpoint` 단계 PASS

---

## 2-3분: 오류 로그 스팟체크

1. Vercel Functions 로그 최근 24시간 확인
2. `500`, `Prisma`, `timeout`, `Unhandled` 키워드로 빠르게 훑기
3. 같은 오류가 3회 이상 반복되면 이슈 1건 생성

완료 기준
- 반복 오류 없음 또는 이슈 생성 완료

---

## 조건부: 관리자 화면 수동 확인

아래 변경이 있을 때만 실행한다.

- 인증/세션
- 관리자 화면
- 신고/제재/정책
- DB migration
- production env

확인 경로:

- `/admin/ops` 접근 및 표시
- 변경한 관리자 큐 1개 표시 또는 저장

## 기록

필요할 때 `docs/reports/`의 별도 증적에 아래 3가지만 기록
- 실행 일시
- `ops:evidence:solo` 로컬 결과 파일 또는 `ops-smoke-checks` run URL + PASS/FAIL
- 조건부 관리자 확인 여부

기록 템플릿
```md
### YYYY-MM-DD solo ops check
- ops-evidence-solo: docs/reports/ops-evidence-<timestamp>.md (PASS/FAIL)
- ops-smoke-checks: <run-url or 미실행> (PASS/FAIL/미실행)
- admin manual: <미실행/실행, 경로>
- issue: <없음 또는 링크>
```

---

## On-demand 확장

- `OPS_BASE_URL=https://townpet.vercel.app pnpm -C app ops:evidence`
- `verify_pg_trgm=true` workflow dispatch
- `verify_sentry=true` workflow dispatch

비고
- 현재 기본 운영은 Sentry 없이도 가능하도록 설계됨
- Sentry는 "필수"가 아니라 "문제 추적이 어려워질 때" 도입
