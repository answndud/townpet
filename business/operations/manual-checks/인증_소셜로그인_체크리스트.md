# 인증/소셜로그인 수동 체크리스트

인증 관련 위험도는 높지만, 실제 OAuth provider와 브라우저 탭 동기화는 자동 테스트만으로 완전히 대체하기 어렵습니다. 이 문서는 preview 또는 production 환경에서 운영자가 직접 확인해야 하는 인증 점검 절차를 정리합니다.

## 사전 조건

- 점검 대상 URL을 정한다.
  - preview 또는 production
- 테스트 계정을 준비한다.
  - 이메일 로그인 가능한 일반 사용자 1개
  - 정지 상태 사용자 1개
  - 운영자 또는 모더레이터 계정 1개
- 카카오/네이버 OAuth 앱의 redirect URL이 현재 대상 환경에 맞게 설정되어 있어야 한다.
- 결과 캡처는 로컬 수동점검 기록 파일에만 남긴다.

## 1. 대소문자 이메일 로그인

1. 이메일 로그인 계정의 실제 저장값이 `user@example.com` 형태인지 확인한다.
2. `/login`에서 같은 이메일을 대문자 변형으로 입력한다.
   - 예: `USER@EXAMPLE.COM`
3. 정상 로그인 후 `/feed` 또는 지정한 `next` 경로로 이동하는지 확인한다.

기대 결과
- 로그인 성공
- 헤더가 즉시 로그인 상태로 전환
- 새로고침 없이도 주요 인증 UI가 정상 상태를 유지

## 2. 멀티탭 로그인/로그아웃 동기화

1. 같은 브라우저에서 TownPet 탭을 2개 연다.
2. 탭 A에서 로그인하고 탭 B를 그대로 둔다.
3. 탭 B에서 새로고침 없이 로그인 상태 UI가 반영되는지 확인한다.
4. 탭 A에서 로그아웃한다.
5. 탭 B에서 새로고침 없이 다시 비로그인 상태 UI로 돌아오는지 확인한다.

기대 결과
- 로그인 후 다른 탭 헤더가 `로그인`에서 `로그아웃`으로 바뀜
- 로그아웃 후 다른 탭 헤더가 다시 `로그인`으로 돌아옴
- 로그아웃 뒤 보호된 페이지 진입 시 재로그인이 요구됨

## 3. 오래된 세션 무효화

1. 로그인 상태에서 브라우저 개발자도구 또는 프록시로 현재 세션 쿠키를 확보한다.
2. 같은 계정으로 로그아웃한다.
3. 확보해 둔 이전 세션 쿠키로 `/api/viewer-shell` 또는 인증이 필요한 API를 다시 호출한다.

기대 결과
- 이전 세션 쿠키는 더 이상 인증되지 않음
- `/api/viewer-shell` 기준 `isAuthenticated: false`

## 4. 정지 계정 로그인 차단

1. 운영자 계정으로 대상 사용자에게 7일 정지 이상 제재를 적용한다.
2. 정지 계정으로 `/login`에서 이메일 로그인을 시도한다.

기대 결과
- 로그인 실패
- 세션 생성 안 됨
- 필요 시 운영 로그 또는 auth audit에서 실패 사유를 확인할 수 있음

## 5. 프로필에서 카카오/네이버 연결

1. 기존 이메일 로그인 계정으로 `/profile`에 들어간다.
2. `계정 연동` 섹션에서 카카오 또는 네이버 연결 버튼을 누른다.
3. OAuth 로그인 완료 후 다시 `/profile`로 돌아오는지 확인한다.

기대 결과
- 프로필 상단 notice에 연결 성공 메시지 노출
- `계정 연동` 섹션에서 해당 provider가 `연결됨` 상태로 바뀜
- 이후 같은 소셜 로그인으로도 동일 계정에 로그인 가능

## 6. OAuthAccountNotLinked 복구 동선

1. 이미 이메일 또는 다른 소셜 방식으로 가입된 계정과 같은 이메일을 가진 카카오/네이버 계정으로 직접 로그인 시도한다.
2. 로그인 실패 시 안내 문구를 확인한다.
3. 안내대로 기존 로그인 방식으로 먼저 로그인한 뒤 `/profile`의 `계정 연동`에서 다시 연결한다.

기대 결과
- 로그인 화면에 "기존 방식으로 먼저 접속 후 프로필에서 연결" 안내가 보임
- 복귀 링크가 `/profile`을 가리킴
- 연결 완료 후 같은 provider로 동일 계정에 로그인 가능

## 7. 프로필에서 카카오/네이버 연결 해제

1. 비밀번호가 설정된 계정 또는 다른 소셜 로그인도 연결된 계정으로 `/profile`에 들어간다.
2. `계정 연동` 섹션에서 이미 연결된 카카오 또는 네이버의 `해제` 버튼을 누른다.
3. 현재 로그인 중인 provider를 해제하는 경우에는 로그인 화면으로 이동하는지 확인한다.

기대 결과
- 마지막 로그인 수단만 남은 상태에서는 `유일한 로그인 수단` 안내와 함께 해제 버튼이 비활성화됨
- 현재 로그인 provider를 해제하면 기존 세션이 무효화되고 로그인 화면으로 이동
- 로그인 화면에 연결 해제 완료 notice가 노출됨
- 해제된 provider로는 더 이상 같은 계정에 로그인되지 않음
