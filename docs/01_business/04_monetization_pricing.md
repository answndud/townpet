# 04. 수익화/가격 전략(가설)

## 1) 수익화 원칙
- 커뮤니티/로컬 서비스의 초기 목표는 “UGC/리텐션”이다.
- 결제 도입은 트랜잭션/CS 비용을 증가시키므로, **부스팅/광고 → 구독 → 트랜잭션 결제** 순서로 진행한다.
- 가격은 “단가 최적화”보다 **전환/유지 가능성**을 우선한다.

## 2) Local BM 상세
### 2.1 지역 타겟 광고(상단 노출/쿠폰)
- 상품(예시)
  - 상단 노출(동네 탭/카테고리별)
  - 업체 프로필 강화(사진/메뉴/전문분야/예약 링크)
  - 쿠폰 발행(사용 조건/기간/수량)
- 결제 단위
  - 월 정액 + 동네/카테고리 가중
  - 또는 CPC/리드(전화/길찾기 클릭 등)

### 2.2 중개 수수료(마이크로 펫시팅/도그워커)
- 기본 구조: 요청 등록 → 매칭 → 완료 → 후기
- 수익화: 성공 수수료(%) 또는 고정 수수료(건당)
- 결제는 Phase 2~3 이후(분쟁/취소/환불 정책 준비 후)

### 2.3 동네 공동구매
- 구성: 공동구매 오픈 → 모집(마감) → 결제 → 배송/픽업
- 수익화: 공동구매 수수료(%) + 배송비 절감으로 사용자 효용 제공
- 리스크: 재고/배송/환불. 초기에는 “공동 주문 링크 + 인원 모집” 수준으로 검증 가능

### 2.4 유기견 홍보 & 봉사 모집
- 목표: 사회적 가치 + 로컬 브랜드 신뢰
- 수익화 직접 모델보다는 파트너십/후원/광고 연계(신뢰 기반)

## 3) Global BM 상세
### 3.1 어필리에이트(제휴)
- 대상: 사료/용품/영양제(알러지/상황 기반 추천)
- 전제: 추천은 “광고”가 아니라 “필터/정보 기반”이어야 신뢰 유지
- KPI: 클릭률, 구매 전환률, 환불/불만 비율

### 3.2 프리미엄 멤버십
- 혜택(예시)
  - 광고 제거
  - 전문가 콘텐츠 풀액세스/요약
  - 전문가 유료 상담권(월 N회)
  - 멤버십 전용 특가/공동구매 우선권
- 가격: 월/연간(연간 할인)

### 3.3 데이터 비즈니스(고급)
- 보험 매칭/제조사 리포트
- 필수 조건
  - 개인정보 최소화/가명처리, 명시적 동의
  - “개별 사용자”가 식별되지 않는 집계/세그먼트 형태
  - 법무/정책 검토 및 투명성 문서(데이터 사용 목적/범위)

## 4) 결제 도입 조건(가이드)
- 마켓/돌봄 거래: 분쟁률(신고/취소) 안정화, 운영 프로세스 확립 후 도입
- 멤버십: Global에서 재방문과 콘텐츠 소비가 확인된 후 도입
- 광고: 로컬 카테고리의 “의도 기반 트래픽”이 충분히 생긴 후 도입
