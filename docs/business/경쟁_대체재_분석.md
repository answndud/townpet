## 1) 결론

TownPet의 실질 경쟁은 특정 앱 하나가 아니라 아래 4가지 역량이다.

1. 구조화 데이터로 의사결정 시간을 줄이는 역량
2. 지역 문맥에서 필요한 사람에게 도달시키는 역량
3. 스팸/분쟁/악용 비용을 자동화로 낮추는 역량
4. 반복 질문을 지식 자산으로 전환하는 역량

## 2) 경쟁 프레임

## 4) 한국 채널 현실 (초기 유입)

- 한국은 채널 침투율이 높아 "노출량"보다 "참여 문맥"이 중요하다.
- 초기 고정 스택:
  - 네이버 검색/블로그: 고의도 획득
  - 카카오 오픈채팅: 질문-답변 전환
  - 인스타 릴스: 상단 퍼널 확장
- 당근 동네생활/X/Threads는 보조 채널로 운영하는 것이 효율적이다.

## 5) 영역별 대체재 대비 포지셔닝

### 5.1 Local 정보

- 대체재 강점: 유저풀, 접근성
- 대체재 약점: 비교 가능한 필드/최신성 관리 약함
- TownPet 포인트: 템플릿 리뷰 + 저장/알림 + 정책 자동화

### 5.2 Market/공동구매

- 대체재 강점: 거래량/익숙한 UX
- 대체재 약점: 분쟁 프로토콜 부재
- TownPet 포인트: 상태 전이, 체크리스트, 제재/신고 정책 내장

### 5.3 Lost&Found

- 대체재 강점: 바이럴 가능성
- 대체재 약점: 반경 타겟팅/종료 프로토콜 부재
- TownPet 포인트: 공용 보드 + 반경 기반 탐색 + 악용 제한

### 5.4 Global 지식

- 대체재 강점: 콘텐츠 총량
- 대체재 약점: 중복 질문/파편화
- TownPet 포인트: 구조화 질문/답변과 커뮤니티 문맥 결합

## 6) TownPet의 방어선

1. Template moat: 데이터 품질 표준화
2. Ops moat: 신고/제재/레이트리밋 자동화
3. Context moat: 지역 + 동물군 + 보드 구조 결합
4. Trust moat: 비회원 포함 동일 정책 집행

## 7) 실행 우선순위 (8주)

| 주차 | 목표 | 필수 실행 | 지표 |
|---|---|---|---|
| 1-2주 | 타깃 고정 | 지역 1개 + 상황 1개 + 템플릿 3종 | 첫 행동 전환율 |
| 3-4주 | 루프 형성 | 시드 콘텐츠 + 답변 품질 관리 | 주간 재방문률 |
| 5-6주 | 신뢰 고정 | 신고 SLA/제재 로그/신규 제한 튜닝 | 신고 24h 처리율 |
| 7-8주 | 수익화 준비 | 제휴 인터뷰 10건 + 슬롯 가설 | 전환 의향률 |

## 8) 리스크와 대응

- 리스크: 카테고리 과확장으로 밀도 하락
  - 대응: 핵심 주제 밀도 달성 후 단계 확장
- 리스크: 성장보다 악용이 빠른 구간
  - 대응: 신규 제한/링크 제한/신고 자동숨김 출시 전 고정
- 리스크: 법적 분쟁(비방/의료 표현)
  - 대응: 문구 정책 + 운영 SLA + 기록 기반 조치

## 9) 실행 체크리스트

1. 새 기능이 Local/Global/공용보드 중 하나로 명확히 귀속되는가?
2. 템플릿 필수 입력과 정책 집행 경로가 있는가?
3. 악용 시나리오 3개와 차단 규칙이 테스트로 고정됐는가?
4. 성장 지표와 안전 지표를 함께 본 뒤 확장하는가?
