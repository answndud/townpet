# 03. MVP 범위(v1) 정의

## 0) 목표
- Phase 1에서 “동네 기반 정보 공유”로 **UGC/리텐션의 핵심 루프**를 만든다.
- Phase 2에서 “거래/번개/긴급” 기능을 붙여 **네트워크 효과**를 강화한다.
- Global은 Phase 2 이후에 본격화하여, Local을 희석하지 않으면서 검색 유입을 확보한다.

---

## Phase 1 (필수 출시 범위)
### 포함(Do)
- Local: 병원/동반가능 장소 템플릿 리뷰
- Local: 산책로 공유(지도, 안전/환경 태그)
- Core: 로그인/프로필/동네 설정
- Core: 게시물/댓글/저장/검색(최소)
- Core: 신고/차단/자동 블라인드(단순 룰)
- 운영: 관리자 신고 큐(최소 기능)

### 제외(Don’t)
- 실시간 채팅(번개/거래는 “댓글/참여” 중심)
- 결제/정산/에스크로
- 전문가 콘텐츠/검수 체계

### 성공 기준(초기 KPI)
- 7일 리텐션(동네 기준)
- 템플릿 리뷰 작성률(방문일/비용/대기 입력 완료 비율)
- 저장/검색 사용률

---

## Phase 2 (로컬 네트워크 강화)
### 포함
- Local: 산책 메이트 번개(모임 생성/참여/취소, 인원 제한)
- Local: 펫친소 프로필 탐색(성향 태그, 동네 친구)
- Market: 중고/대여/나눔(상태 변경, 인수/반납 체크)
- Local: 실종/유기 신고(게시 + “긴급” 구독자 알림)

### 제한(의도적 단순화)
- 거래/번개 메시징은 내부 DM 대신 “알림 + 댓글/연락 방식 제한”으로 시작
- 실종/유기 푸시는 “동네 스코프 + 반경 기반”만 제공(정밀 제어는 후순위)

---

## Phase 3 (Global 지식 & 돌봄 수익 모델 준비)
### 포함
- Global: 품종 토픽, Q&A 채택
- Global: 사료/용품 DB(기본) + 리뷰
- Local: 마이크로 펫시팅(요청/수락/완료/후기) — 결제는 옵션
- BM: 제휴 링크(사료/용품) 도입

---

## Phase 4~ (결제/광고/멤버십 고도화)
- 결제/정산/수수료, 광고/부스팅, 공동구매, 멤버십
