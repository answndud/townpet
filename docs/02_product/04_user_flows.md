# 04. 사용자 플로우(핵심 시나리오)

## 1) 온보딩/동네 설정
1. 로그인(OAuth)
2. 닉네임/프로필 설정
3. 대표 동네 선택(지도/검색)
4. (선택) 동네 인증 레벨 업그레이드
5. 홈 진입(로컬 피드)

---

## 2) 동물병원 리뷰(템플릿)
1. Local > 병원/장소 > “리뷰 작성”
2. 병원 선택(검색/직접 입력)
3. 템플릿 입력
   - 방문일(필수), 진료 항목 태그(필수), 비용 총액(필수), 대기시간(필수)
   - 영수증 여부(체크), 항목별 금액(선택)
4. 게시
5. 다른 사용자는 저장/댓글/신고 가능
6. 병원 프로필에서 리뷰가 누적되어 평균/분포가 계산됨

예외/운영
- 비방 문구 포함 → 작성 시 경고 → 게시 후 신고 누적 시 자동 블라인드

---

## 3) 동반 가능 장소(식당/테라스 카페) 공유
1. 장소 리뷰 작성
2. 조건 태그 선택(실내/테라스, 케이지 필요, 대형견 가능 등)
3. 사진 업로드(선택)
4. 지도에서 조회/필터

---

## 4) 산책로 공유(지도)
1. Local > 산책로 > “코스 등록”
2. 시작점/경로(폴리라인) 입력(초기에는 시작점+경유지로 단순화 가능)
3. 태그 입력(가로등/화장실/차량/결빙 등)
4. 게시
5. 근처 사용자에게 추천(거리 기반)

---

## 5) 산책 메이트 번개(모임)
1. Local > 번개 > “지금 모임 만들기”
2. 장소(공원) + 시간(즉시/예약) + 인원수 + 조건(대형견/사회성 등)
3. 참여 신청
4. 주최자 승인(선택) 또는 선착순
5. 취소/노쇼 처리(후기/제재 정책과 연결)

---

## 6) 마켓(중고/대여/나눔)
### 6.1 등록
1. 마켓 > “등록”
2. 유형 선택(중고/대여/나눔)
3. 필수 입력
   - 카테고리, 가격(중고), 대여료/보증금/기간(대여), 상태, 위치
4. 게시

### 6.2 문의/상태 변경
- 구매/대여 문의 → 알림 발송
- 판매자: 상태 변경(판매중→예약→완료)
- 대여: 인수/반납 사진 체크(분쟁 최소화)

---

## 7) 심부름/긴급 돌봄(마이크로 펫시팅)
1. 돌봄 > “요청 등록”
2. 유형(산책/급식/병원 동행/긴급 체크) + 시간 + 위치 + 보상(선택)
3. 주변 사용자에게 노출/알림
4. 제공자 수락 → 수행 → 완료
5. 후기/평점 → 신뢰도 반영
(결제는 Phase 4 이후 가능)

---

## 8) 실종/유기 긴급 알림
1. Local > 실종/유기 > “긴급 신고”
2. 필수 정보
   - 사진, 특징, 마지막 위치/시간, 연락 방법(노출 제한 옵션)
3. 게시 즉시 “긴급” 처리
4. 반경/동네 기준으로 푸시/알림 전송(구독자 + 인근 사용자)
5. 제보 등록(위치/시간/사진)
6. 종료 처리(찾음/보호소 인계) + 감사 메시지(선택)

운영/리스크
- 허위 신고 방지: 제한(가입 기간/동네 인증/신고 이력), 악용 시 제재
