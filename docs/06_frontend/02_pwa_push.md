# 02. PWA/푸시(웹 기준) 설계

## 1) PWA 목표
- 설치 장벽 없이 “앱처럼” 사용
- 핵심은 알림/바로가기/캐시

## 2) PWA 필수 구성
- `manifest.json`(아이콘, 색상, 시작 URL)
- Service Worker(기본 캐시)
- 홈 화면 추가 안내(특히 iOS)

## 3) 웹 푸시 정책
- Android/Chrome 환경에서 우선 지원
- iOS는 환경 제약이 있으므로 이메일 알림을 보조 채널로 둔다.

## 4) 알림 유형
- 댓글/답글
- 저장 키워드 새 글
- 번개 참여/변경
- 마켓 문의/상태 변경
- 실종/유기 긴급(최우선)

## 5) 앱 확장 대비
- 푸시 토큰 저장 구조를 서버에 마련(FCM/APNs로 확장)
- 딥링크 규칙: `/p/{postId}` 형태 등
