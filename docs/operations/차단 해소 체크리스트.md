# Blocked 해소 체크리스트 (OAuth 실E2E + Sentry 실수신)

기준일
- 2026-02-24

목적
- `PLAN.md`의 환경 의존 blocked 항목을 운영자가 빠르게 해소할 수 있도록 필수 시크릿/설정/실행/판정 기준을 한 문서로 고정한다.

## 1) OAuth 실계정 E2E 해소

대상 워크플로우
- `.github/workflows/oauth-real-e2e.yml`

필수 GitHub Secrets
- `AUTH_SECRET` 또는 `NEXTAUTH_SECRET` (둘 중 하나 이상)
- `KAKAO_CLIENT_ID`
- `KAKAO_CLIENT_SECRET`
- `NAVER_CLIENT_ID`
- `NAVER_CLIENT_SECRET`

사전 콘솔 설정
- Kakao/네이버 OAuth Redirect URI에 아래 값 등록
  - `http://localhost:3000/api/auth/callback/kakao`
  - `http://localhost:3000/api/auth/callback/naver`
- 운영 점검 전 `pnpm -C app ops:oauth:preflight` 실행 후 Base URL sanity `ERROR`가 없는지 확인

실행 순서
1. GitHub Actions > `oauth-real-e2e` > `Run workflow`
2. 잡 `oauth-real-smoke` 실행
3. 실패 시 `Validate OAuth secrets` 단계에서 누락 시크릿 이름 확인

실계정 로그인 완료 수동 점검(카카오/네이버 각각 1회)
1. 시크릿/프라이빗 브라우저에서 `https://townpet.vercel.app/login?next=%2Fonboarding` 접속
2. 카카오(또는 네이버) 로그인 버튼 클릭 후 공급자 로그인 완료
3. `/onboarding` 진입 확인 후 닉네임 저장 + 대표 동네 1개 저장
4. `나중에 설정하기` 클릭 또는 온보딩 완료 후 `/feed` 도달 확인
5. 성공 증적 1개 이상 확보(스크린샷 또는 짧은 화면 녹화)
6. 동일 날짜에 카카오/네이버 각각 수행 결과를 `PROGRESS.md`에 기록
7. 수동 점검 템플릿 파일(`docs/operations/manual-checks/oauth-manual-check-<YYYY-MM-DD>.md`)에 증적 링크/상태를 채워 보관

완료 판정(DoD)
- `Real OAuth redirect smoke` 단계 PASS
- `app/e2e/social-real-oauth-redirect.spec.ts` 실행 결과 PASS
- 실계정 수동 점검에서 카카오/네이버 모두 `/onboarding -> /feed` 완료 증적 확보

## 2) Sentry 실수신 검증 해소

대상 워크플로우
- `.github/workflows/ops-smoke-checks.yml`

필수 GitHub Secrets
- `SENTRY_DSN`
- `SENTRY_AUTH_TOKEN`
- `SENTRY_ORG_SLUG`
- `SENTRY_PROJECT_SLUG`

사전 입력값
- `target_base_url`: 검증할 실제 배포 URL (예: `https://townpet.vercel.app`)
- `verify_sentry`: `true`

실행 순서
1. GitHub Actions > `ops-smoke-checks` > `Run workflow`
2. `target_base_url` 입력 + `verify_sentry=true`
3. `Check deployment health endpoint` PASS 확인
4. `Prewarm guest feed and search routes` PASS 확인
5. `Check Sentry ingestion` PASS 확인

주간 자동 점검(health only)
1. `.github/workflows/ops-smoke-checks.yml`의 `schedule` 실행 사용(매주 월요일 UTC 00:15)
2. 자동 실행은 `verify_sentry=false` 경로(health only)로 동작
3. 필요 시 수동 실행으로 `verify_sentry=true`를 추가 점검

완료 판정(DoD)
- `pnpm ops:check:health` PASS
- `pnpm ops:prewarm` PASS
- `pnpm ops:check:sentry` PASS
- 스크립트 출력에 Sentry event id 확인 로그 존재

## 3) 실패 시 빠른 트리아지

OAuth 실패
- 시크릿 누락: workflow 로그 `Missing required secret` 확인 후 Secrets 추가
- provider redirect mismatch: Kakao/Naver 콘솔 Redirect URI 재확인
- callback mismatch:
  - Base URL이 `https://<운영도메인>`인지 확인
  - `vercel.com`, `*-projects.vercel.app` 도메인 사용 금지(SSO 로그인으로 이탈 위험)
  - Kakao/Naver 콘솔의 callback URL이 `/api/auth/callback/{provider}`와 완전 일치하는지 확인

Sentry 실패
- DSN/토큰 오타 또는 권한 범위 부족 여부 확인
- org/project slug 불일치 여부 확인
- 배포 URL 오입력/health endpoint 응답 코드 확인
